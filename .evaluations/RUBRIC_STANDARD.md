# Sophia 评测标准 v2.1

> 这是 Sophia Rubric Lab 的**评测标准说明书**，面向使用者。
> 它定义了"我们如何判断一份 AI 产品报告的好坏"。
> 网站「标准」tab 直接渲染此文件 —— 改它即改标准，刷新即见最新。
>
> **版本**：v2.1（2026-04-22 生效，基于 v2 的增量修订）
>
> **v2.1 相对 v2 的变化**（评测流程与产物要求升级，维度权重不变）：
> - R1 新增**「外部核验」硬约束**：评测官必须主动发起外部搜索/计算核验，不能只靠报告内部自洽判断准确性
> - R2 新增**「潜在问题挖掘」子项**：要求识别 query 字面下的真问题、深层诉求
> - 评测产物新增**「per-report feedback」结构化字段**（每份报告的"做得好/做得不好/改进建议"三件套）
> - 评测报告正文**硬性要求**：事实性错误、逻辑性错误必须逐条详细展开（见 EVALUATION_CONTRACT §3.5）
>
> **v2 相对 v1 的主要变化**：
> - 5 大维度重构（信源并入准确性、相关性独立、论证深度强化、决策价值重写）
> - 准确性权重拉到 0.40，引入「一票否决」硬规则
> - 打分从 0.5 精度改为 5 档（S/A/B/C/D），不许小数
> - 扩展维度升级为"一等公民"，允许激活一个纳入总分

---

## 一、评测宗旨

我们做的是**「AI 产品对 AI 产品」的横向评测**。

针对同一个 Query（商业、投研、技术、生活等真实问题），让多个 AI 产品（SophiaAI、MiroThink、ChatGPT、DeepSeek、Claude 等）各自生成一份报告，然后由评测官 Sophia 打分 + 写评测报告 + 给出 SBS 结论。

**我们要回答的核心问题是：**

> **这份报告，能不能真的帮到一个要做决策的人？**

这意味着我们衡量的是：

- 说的东西**对不对**（事实、数字、信源）
- 答的是不是**用户真正问的问题**
- 有没有**真正的分析和推导**，而不只是信息罗列
- 是否覆盖**决策所需的关键维度**
- 看完之后用户是否**多知道了什么有用的东西**、能否**帮到下一步决策**

**我们不衡量的：**

- 篇幅长度
- 排版漂亮程度（markdown 花哨度不计分）
- 语气是否讨好
- 术语堆砌感

## 二、五大维度（Rubric v2）

这 5 个维度是评测的骨架，**每一份评测都必须覆盖全部 5 项**，权重固定。

| ID | 名称 | 权重 | 一句话定义 |
|----|------|------|---------------|
| **R1** | **准确性**（Accuracy） | **0.40** | 事实、数据、口径、信源是否都经得起核查——**生命线** |
| **R2** | **相关性**（Relevance） | **0.15** | 是否正面回应用户 query、是否围绕用户真实诉求展开 |
| **R3** | **论证深度**（Reasoning） | **0.20** | 观点有证据、推导闭环、分析有深度、能呈现非共识视角 |
| **R4** | **完备性**（Completeness） | **0.10** | 决策所需的关键维度是否齐全、结构是否支撑论证 |
| **R5** | **决策价值**（Decision Value） | **0.15** | 是否真正帮到用户、有无信息熵增量、有无启发、有无可操作 take-away |

权重加总 = 1.00。

---

### R1 · 准确性（Accuracy）· 权重 0.40

**核心问：这份报告说的东西，对不对？支撑它说的证据，硬不硬？**

准确性是评测的**生命线**——也是 v2 相对 v1 最大的变化：我们把"数据对不对"和"信源硬不硬"合并到同一个维度，并给它最高权重，因为两者都指向同一件事——**报告的事实基础是否经得起核查**。

#### 判定子项（MECE）

| 子项 | 判什么 | 怎么看 |
|---|---|---|
| **事实准确** | 陈述的事件、人物、因果、定性判断是否真实成立 | 逐条可核查的事实命题，交叉外部信源 |
| **数字准确** | 数值、金额、比例、日期、单位、量级是否正确 | 关键数字对照一手来源 |
| **口径统一** | 同一指标是否用同一口径（营收含税/不含税、净利/扣非、YoY/QoQ），跨段落是否一致 | 看口径声明 + 跨段落交叉 |
| **语境准确** | 数据 / 引用 / 事件是否放在对的时间 / 范围 / 主体上，无张冠李戴、无时空错位 | 看上下文匹配 |
| **信源可核查** | 关键事实和数字是否有可追溯的引用（URL / 文档 / 公告），是否一手、是否层级清晰 | 看引用规范度 + 一手源占比 |

#### 🔎 外部核验要求（v2.1 新增硬约束）

R1 准确性**不能仅靠"报告内部自洽"判定**。评测官必须主动走外部核验路径：

1. **关键数字抽样**：每份报告里选 3~5 条"承重数字"（承载核心结论的数字），逐条对照一手源核验
2. **硬伤怀疑必查**：只要评测官对某条事实/数字**产生怀疑**（数字看起来过于精确、与常识偏差大、没有任何信源支撑、精确到亿但无出处等），**必须**发起外部搜索或逻辑/量纲计算交叉核验
3. **一票否决必须外验**：任何 veto 候选（疑似量级错/主体错/时间错/编造信源）**写结论前必须至少做一次外部验证**，不能仅凭"报告内没给信源"就下 veto 判决
4. **口径自校**：量纲、占比、汇率换算等可用算术直接核验的，评测官**必须动手算一遍**

外部核验的结果要落到评测报告正文的「事实核验记录」章节（见 §EVALUATION_CONTRACT.md §3.5）。

#### 🚨 一票否决规则（硬约束）

**R1 若存在「重大事实错误」，无论其他维度打多高，最终评级锁死在「合格」档上限，总分封顶 6.9。**

"重大事实错误"的判定标准（满足任一即触发）：

- 关键数字**量级错误**（例："600 亿美元"写成"600 万美元"）
- 关键主体**搞错**（把 A 公司的业务说成 B 公司的）
- 关键时间**错位**（引用已失效 2 年的政策当现行）
- 关键因果**颠倒**（"A 导致 B"写成"B 导致 A"）
- **编造信源**（引用了根本不存在的文献/公告）

"关键"指的是：**这条信息如果用户相信了会影响决策**。非关键位置的小错（侧枝数字小数点位错误、人名写错但主体正确）只在 R1 内扣档，不触发一票否决。

### R2 · 相关性（Relevance）· 权重 0.15

**核心问：用户问什么，报告是不是正面答了什么？**

这是 v2 新独立的维度——原先散落在 v1 的 R4/R5，现在显式化。一份报告可以写得很漂亮、数据很准，但如果**答非所问**，对用户来说就是零价值。

#### 判定子项

| 子项 | 判什么 |
|---|---|
| **正面回应** | 用户 query 的核心诉求是否被**直接回答**（不绕弯、不避重就轻、不答非所问） |
| **诉求覆盖** | query 里多个子问题 / 隐含诉求是否都被触达 |
| **锚点命中** | query 里的关键锚点（时间 / 地域 / 主体 / 场景）是否被正确识别并响应（例如 query 说"最新时政"，报告是否真给了时政锚点） |
| **潜在问题挖掘** | 能否识别出 query 字面之下的**真问题/深层诉求**，并主动回应（例如用户问"值不值得买"，真问题可能是"风险-收益是否匹配我的持有周期"；用户问"A 和 B 谁强"，真问题可能是"在我的场景下选哪个"）|
| **内容贴题** | 正文是否围绕 query 展开，无跑题、无大段无关填充 |

> **"潜在问题挖掘"子项说明**：这是 v2.1 新增子项，用来奖励那些**不止于回答字面问题，还能帮用户把问题想深一层**的报告。如果报告只是机械地按字面 query 应答，没有对潜在诉求的回应，这一子项得分会压制 R2 档位；反之，如果报告主动点出"你真正想问的可能是 X"并且给出对应的分析，R2 档位应相应上抬。判定时要看报告是否体现出对 query 背后决策场景的理解。

**典型反例**（自动扣档）：

- query 问"XX 公司最新一季财报"，答的却是"XX 行业综述"
- query 要"最新时政锚点"，报告通篇无时政相关内容
- query 要"对比 A 和 B"，报告只讲了 A
- query 隐含"我在 XX 场景下做决策"，但报告通篇只做字面应答、没有任何对决策场景的回应

---

### R3 · 论证深度（Reasoning）· 权重 0.20

**核心问：报告不只是在"罗列"，而是在"分析"——观点有证据，推导能闭环，甚至能看到别人看不到的东西。**

#### 判定子项

| 子项 | 判什么 |
|---|---|
| **论证可支撑** | 每一个观点 / 判断是否有证据或推导支撑，不是空泛断言；论据 → 结论链条是否闭环，无逻辑跳跃 |
| **分析深度** | 有没有"为什么"的追问，而不只是"是什么"的罗列；有没有追到二阶、三阶原因 |
| **非共识信号** | 有没有独家洞察、反直觉但成立的判断、差异化视角——**这是深度研究区别于信息堆砌的关键** |
| **多视角平衡** | 是否呈现对立视角 / 反方证据，而不是单方面站队 |

> R1 与 R3 的 MECE 分工：
> - **R1 问**："你说的这个数字/事实对不对？" → 事实本身对错
> - **R3 问**："你基于这个事实得出的结论合不合理？" → 推理链条强度

---

### R4 · 完备性（Completeness）· 权重 0.10

**核心问：从决策视角看，关键维度是不是都覆盖了？结构能不能支撑论证？**

#### 判定子项

| 子项 | 判什么 |
|---|---|
| **维度齐全** | 该考虑的主要分析维度是否都覆盖（例如投资报告必备的基本面/竞争/风险；行业综述必备的市场/技术/政策） |
| **结构合理** | 章节安排是否支撑结论、有无重复/空白、是否有清晰的论证骨架 |
| **定量深度** | 是否有必要的定量测算 / 表格 / 模型支撑（适用时） |

---

### R5 · 决策价值（Decision Value）· 权重 0.15

**核心问：这份报告看完之后，用户比看之前多知道了什么有用的东西？能不能帮他做下一步决定？**

R5 在 v2 里被重写——核心诉求是**信息熵增量 + 启发性 + 可决策性**。一份报告即使 R1~R4 都完美，如果只是把公开信息重新组装一遍，对用户就没有真正的价值。

#### 判定子项

| 子项 | 判什么 |
|---|---|
| **信息增量** | 是否给出用户自己搜不到 / 想不到的信息？还是只是把公开信息重新组装一遍？ |
| **启发性** | 是否提供了新角度、新框架、新联想，让用户对问题的理解**有跃迁**？ |
| **可操作 take-away** | 是否给出具体、清晰、可执行的建议或结论，让用户"知道接下来该做什么"？ |
| **风险与不确定性诚实度** | 是否主动点出 caveat、不确定项，诚实标注"这里我没把握"？ |

**典型反例**（自动扣档）：

- 通篇都是维基百科级别的公开信息重述，零增量
- 只有"总体向好 / 需警惕风险"这种废话型结论，零启发
- 给出"建议谨慎关注"这种零操作性的敷衍 take-away
- 对明显不确定的信息打包票，不给 caveat

## 三、评分机制（档位打分法）

抛弃 v1 的"0-10 × 0.5 精度 + 反向凑分"伪精度游戏。v2 采用**档位打分**：

### 5 档制

每个维度**只允许**从下面 5 个档位中选一个：

| 档位 | 分数 | 含义 |
|---|---|---|
| **S** | **10** | 卓越 —— 业内最强水平，挑不出毛病 |
| **A** | **8** | 优秀 —— 明显高于平均，可直接交付 |
| **B** | **6** | 合格 —— 基本可用但有明显短板 |
| **C** | **4** | 待改进 —— 有显著缺陷 |
| **D** | **2** | 不合格 —— 结构性问题，不可用 |

**硬约束：**

- 每个维度的 score 只能是 **10 / 8 / 6 / 4 / 2** 中的一个整数
- 不允许 0.5、0.1 之类的小数
- 不允许 9、7、5、3、1 之类"档间分"

### 总分计算

```
overallScore = Σ(Ri.score × Ri.weight)
```

权重 40/15/20/10/15，所以一般落点例如：

| 组合（R1/R2/R3/R4/R5） | 总分 | 评级 |
|---|---|---|
| S/S/S/S/S（10×5） | 10.0 | 🏆 卓越 |
| A/A/A/A/A（8×5） | 8.0 | ⭐ 优秀 |
| S/A/A/A/A | 8.8 | 🏆 卓越 |
| A/A/S/A/A | 8.4 | ⭐ 优秀 |
| A/A/A/B/A | 7.8 | ⭐ 优秀 |
| B/A/A/A/A | 7.2 | ⭐ 优秀 |
| B/A/S/A/S | 7.5 | ⭐ 优秀 |
| C/A/A/A/A | 6.4 | ✅ 合格 |
| 一票否决触发（R1 重大错误） | 封顶 6.9 | ✅ 合格（上限） |

**算完即定，不反向凑分**。总分是直接从维度分数加权出来的结果，不要再为了"凑整"去改维度分。

### 评级档位

| 总分区间 | 评级 | 含义 |
|---------|------|------|
| 8.5 ~ 10.0 | 🏆 **卓越** | 接近无可挑剔，可以直接交付决策 |
| 7.0 ~ 8.4 | ⭐ **优秀** | 质量明显高于大多数，需少量修订即可用 |
| 5.5 ~ 6.9 | ✅ **合格** | 基本可用，但存在可见短板 |
| 4.0 ~ 5.4 | ⚠️ **待改进** | 有明显缺陷，不建议直接采用 |
| 0 ~ 3.9 | ❌ **不合格** | 结构性问题，需要重写 |

> 一票否决规则生效时：无论维度分数如何，总分封顶 6.9，评级封顶"合格"。

### 辅助字段（不影响总分）

每个维度的分数旁可以附带两个辅助字段：

- **`confidence`**：`high` / `medium` / `low` —— 评测官对这个打分的确信度（客观可数的高，主观判断的中，几乎全凭感觉的低）
- **`issueTags`**：问题标签数组（见第五节词表）

这些字段只影响展示，不参与总分计算。

## 四、扩展维度（Extra Dimensions）

v2 里扩展维度从"可选补丁"升级为**"一等公民"**——我们鼓励评测官在 R1~R5 之外**主动发散**，去发现 query 特有的差异信号。

### 使用原则

- **每次评测至少尝试一次**——要么列出 1~3 个扩展维度，要么显式声明"本次无必要扩展"并说明原因
- 每个扩展维度必须提供 `rationale`（为什么这个 query 需要这个维度）
- 扩展维度数量上限 3 个（防止泛滥）

### 激活规则（选做，最多 1 个）

评测官可以选择**激活其中 1 个 X 维度**纳入总分。激活规则：

- 激活权重可选：**0.05 / 0.10 / 0.15** 三档
- R1~R5 原权重按比例等比缩减，保持总和 1.0
- 激活的 X 维度同样使用 5 档制（10/8/6/4/2）
- 未激活的 X 维度不计入总分，只作为开放观察项

**举例**：评测官认为"本地化适配"对这个 query 具有决策级重要性，激活 X1 权重 0.10：

```
原权重：R1/R2/R3/R4/R5 = 40/15/20/10/15
缩减后：R1/R2/R3/R4/R5 = 36/13.5/18/9/13.5 + X1=10  （总和仍为 100）
```

### 扩展维度候选词表（参考，可自拟）

`本地化适配` · `代码正确性` · `创意密度` · `情感共鸣` · `合规敏感度` · `时效敏感度` · `交互友好度` · `技术细节颗粒度` · `数据可视化质量` · `阅读成本` · `风格契合度`

评测官也可以按 query 情况自拟维度名，但要在 rationale 里说清楚为什么需要这个维度。

## 五、常见问题标签（issueTags）

评测官在单维度打分时，可以给每份报告打若干「问题标签」，便于后续聚合分析：

`未标注信源` · `一手源不足` · `数据过期` · `口径混乱` · `数字量级错` · `事实错误` · `编造信源` · `跑题` · `未答核心诉求` · `锚点缺失` · `逻辑跳跃` · `论据缺失` · `结论空泛` · `信息零增量` · `启发性不足` · `缺乏可操作建议` · `风险缺失` · `定量缺失` · `维度缺漏` · `术语误用` · `本地化缺失`

也可以用表外的 tag，但优先使用上述词表以便网站聚合。

## 六、SBS 规则（Side-by-Side 对比）

**对比粒度：AI 产品对 AI 产品**（不是报告对报告；即使同一产品提交了多份报告，也先聚合到产品层再两两对比）。

**对数量：**

| 产品数量 | SBS 对数 |
|---------|---------|
| 2 个 | 1 对 |
| 3 个 | 3 对（两两组合） |
| 4 个 | 6 对 |

**每对输出：**

- **winner**：A / B / 平局
- **margin**：压倒性 · 明显优势 · 略微领先 · 势均力敌
- **keyReason**：一句话点出关键差异维度（例如"A 在 R1/R3 全面领先，R5 可操作 take-away 更强；B 虽然 R2 相关性更贴但 R1 有量级错误触发一票否决"）

## 七、迭代原则

- **一套 rubric，永远最新**：直接编辑本文件 → 网站刷新可见
- 权重调整、维度增删、评级档位修改 —— 都在这里改
- 评测官 Sophia 在对话框里执行评测前，会读工作契约（`EVALUATION_CONTRACT.md`），契约里引用本标准作为打分依据
- 每次标准有实质改动（维度/权重/打分机制），应同步升级 `EVALUATION_CONTRACT.md` 的 `contractVersion`

## 附录：v1 → v2 变更一览

| 项 | v1 | v2 |
|---|---|---|
| 维度定义 | R1 信源真实性 / R2 结构定量 / R3 洞察论证 / R4 风险决策 / R5 专业时效 | R1 准确性 / R2 相关性 / R3 论证深度 / R4 完备性 / R5 决策价值 |
| R1 范围 | 信源 + 数据真实性 | **信源 + 数据 + 事实 + 口径 + 语境**（完全合并） |
| R1 权重 | 0.25 | **0.40** |
| R2 | 结构与定量 | **相关性**（独立）；结构与定量移入 R4 |
| R5 | 专业度与时效 | **决策价值**（信息增量/启发/可操作）；术语准确移入 R1，时效移入 R2 |
| 打分精度 | 0~10，0.5 精度 | **10/8/6/4/2 五档制**（不许小数） |
| 一票否决 | 无 | **R1 重大事实错误 → 总分封顶 6.9** |
| 扩展维度 | 可选，永远不计入总分 | **一等公民，允许激活 1 个纳入总分** |
| 总分精度约束 | overallScore 与加权和 ≤ ±0.1 | 直接等于加权和（档位制下加权和天然是有限精度） |
